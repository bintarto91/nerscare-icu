# Draft Serah Terima dan Maintenance

## 1. Nama Pekerjaan

Pengembangan Web AI-Assisted Assessment ICU Loneliness.

## 2. Pihak

Pihak pertama: Tim peneliti/klien.

Pihak kedua: Developer.

## 3. Ruang Lingkup Serah Terima

Developer menyerahkan:

- Source code aplikasi.
- Database migration.
- Aplikasi yang sudah terpasang di server.
- Akun admin awal.
- Dokumentasi teknis.
- Manual book pengguna.
- RAB dan timeline.
- Checklist UAT.

## 4. Fitur yang Diserahkan

- Landing page.
- Kalkulator loneliness public.
- Login pengguna.
- Dashboard.
- Data pasien.
- Assessment De Jong Gierveld 11 item.
- Hasil dan riwayat assessment.
- Emotional loneliness dan social loneliness.
- Cetak laporan.
- Edukasi perawat.
- Edukasi keluarga.
- Manajemen konten.
- Manajemen pertanyaan.
- Pengaturan interpretasi.
- Pengaturan landing.
- Pengaturan laporan.
- Booklet landing admin-editable.
- Manajemen pengguna.

## 5. Catatan Kondisi Serah Terima

Sistem diserahkan dalam kondisi siap demo/soft live. Untuk live real, disarankan dilakukan:

- Proofread konten.
- UAT pengguna.
- Backup database.
- Validasi etik dan izin penelitian.
- Penetapan SOP penggunaan data pasien.

## 6. Maintenance

Maintenance dapat mencakup:

- Perbaikan bug ringan.
- Bantuan upload revisi.
- Clear cache dan pengecekan server ringan.
- Penyesuaian teks/konten minor.
- Bantuan restore backup jika tersedia.

Maintenance tidak mencakup:

- Pembuatan fitur besar baru.
- Integrasi dengan sistem rumah sakit.
- Migrasi server besar.
- Desain ulang total.
- Pengolahan statistik penelitian.
- Penulisan artikel ilmiah.

## 7. SLA Support yang Disarankan

| Prioritas | Contoh Masalah | Target Respons |
|---|---|---|
| Tinggi | Tidak bisa login, assessment error, data tidak tersimpan | 1 x 24 jam |
| Sedang | Tampilan laporan kurang rapi, konten tidak tampil | 2 x 24 jam |
| Rendah | Perubahan teks, warna, label | 3-5 hari kerja |

## 8. Backup dan Akses

Klien perlu menyimpan:

- Akses server.
- Akses panel hosting.
- Akses database.
- File `.env`.
- Akun admin.

Developer tidak bertanggung jawab atas kehilangan data apabila tidak ada backup atau akses server diubah tanpa pemberitahuan.
