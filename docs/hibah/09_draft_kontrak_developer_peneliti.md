## 2. Latar Belakang

Pihak Pertama membutuhkan aplikasi web untuk mendukung kegiatan hibah penelitian terkait assessment loneliness pasien ICU. Pihak Kedua memiliki kemampuan teknis untuk mengembangkan aplikasi web sesuai kebutuhan Pihak Pertama.

Para Pihak sepakat untuk bekerja sama dalam pengembangan aplikasi web **AI-Assisted Assessment ICU Loneliness** dengan ketentuan dalam dokumen ini.

## 3. Ruang Lingkup Pekerjaan

Pihak Kedua bertanggung jawab mengembangkan aplikasi web dengan ruang lingkup sebagai berikut:

1. Landing page public.
2. Kalkulator loneliness public.
3. Login pengguna.
4. Dashboard.
5. Manajemen data pasien.
6. Assessment loneliness berbasis De Jong Gierveld Loneliness Scale 11 item.
7. Perhitungan total skor, kategori umum, emotional loneliness, dan social loneliness.
8. Hasil dan riwayat assessment.
9. Follow-up/tindak lanjut assessment.
10. Cetak laporan hasil assessment.
11. Edukasi perawat.
12. Edukasi keluarga.
13. Manajemen konten edukasi.
14. Pengaturan pertanyaan assessment.
15. Pengaturan interpretasi.
16. Pengaturan landing page.
17. Pengaturan laporan.
18. Booklet digital landing yang dapat dikelola oleh admin.
19. Manajemen pengguna.
20. Deployment awal ke server.
21. Dokumentasi teknis, manual book, dan checklist UAT.

## 4. Batasan Pekerjaan

Ruang lingkup pekerjaan tidak termasuk:

1. Pengurusan ethical clearance.
2. Pengurusan izin rumah sakit/institusi.
3. Validasi ilmiah instrumen oleh ahli.
4. Proofread akademik final oleh ahli substansi.
5. Pengolahan statistik hasil penelitian.
6. Penulisan artikel ilmiah atau laporan penelitian akhir.
7. Integrasi dengan SIMRS atau sistem eksternal lain.
8. Biaya domain, hosting, VPS, SSL berbayar, atau lisensi pihak ketiga jika belum disediakan.
9. Perubahan fitur besar di luar ruang lingkup tanpa kesepakatan tambahan.

## 7. Timeline Pekerjaan

Estimasi durasi pekerjaan adalah **6 minggu kerja** sejak termin pertama diterima dan kebutuhan utama sudah disepakati.

Timeline dapat berubah apabila:

- Pihak Pertama menambah fitur di luar scope.
- Terdapat perubahan desain/fitur besar.
- Server/domain/akses belum tersedia.
- Materi penelitian/konten belum diberikan.
- Terdapat kendala teknis di server pihak ketiga.

## 8. Kewajiban Pihak Pertama

Pihak Pertama berkewajiban:

1. Memberikan kebutuhan sistem dan revisi secara jelas.
2. Memberikan materi penelitian, instrumen, booklet, edukasi, dan identitas laporan.
3. Menyediakan akses server/domain/database apabila aplikasi dipasang di server Pihak Pertama.
4. Melakukan review dan UAT sesuai jadwal.
5. Memberikan persetujuan final terhadap isi kuesioner, interpretasi, edukasi, dan booklet.
6. Menanggung tanggung jawab etik, izin penelitian, dan perlindungan data pasien.
7. Melakukan pembayaran sesuai termin.

## 9. Kewajiban Pihak Kedua

Pihak Kedua berkewajiban:

1. Mengembangkan aplikasi sesuai ruang lingkup.
2. Menjaga kerahasiaan akses server dan data yang diberikan.
3. Melakukan deployment awal sesuai kebutuhan.
4. Memperbaiki bug yang ditemukan pada masa UAT.
5. Menyerahkan dokumentasi teknis dan manual penggunaan.
6. Memberikan penjelasan penggunaan sistem kepada Pihak Pertama.

## 10. Revisi

Pekerjaan mencakup revisi minor yang berkaitan dengan:

- Perubahan teks.
- Penyesuaian warna/tampilan ringan.
- Perbaikan bug.
- Penyesuaian alur kecil yang masih sesuai scope.

Revisi besar yang memerlukan addendum:

- Penambahan modul baru.
- Perubahan struktur database besar.
- Integrasi sistem eksternal.
- Perubahan total desain.
- Perubahan metode scoring setelah sistem selesai.
- Permintaan fitur statistik/analisis penelitian lanjutan.

## 11. UAT dan Penerimaan

Pihak Pertama melakukan UAT menggunakan checklist yang disediakan. Sistem dianggap diterima apabila:

1. Modul utama dapat digunakan.
2. Tidak ada bug prioritas tinggi.
3. Alur assessment berjalan.
4. Hasil dan laporan dapat ditampilkan.
5. Admin dapat mengelola konten utama.
6. Booklet dapat dikelola dari dashboard.

Jika terdapat catatan, Para Pihak membuat daftar revisi dan target penyelesaian.

## 12. Maintenance

Maintenance setelah serah terima tidak termasuk dalam nilai pengembangan kecuali disepakati terpisah.

Pilihan maintenance yang disarankan:

| Paket | Biaya | Durasi | Keterangan |
|---|---:|---:|---|
| Ringan | Rp500.000/bulan | 3 bulan | Bug fixing kecil dan bantuan operasional ringan |
| Sedang | Rp750.000/bulan | 3 bulan | Bug fixing, monitoring ringan, revisi minor |
| Penuh | Rp1.200.000/bulan | 3 bulan | Support prioritas dan revisi fitur kecil |

Rekomendasi untuk proyek hibah penelitian: **maintenance sedang selama 3 bulan**.

## 13. Hak Kekayaan Intelektual dan Source Code

Setelah pembayaran selesai:

1. Pihak Pertama berhak menggunakan aplikasi untuk kebutuhan penelitian.
2. Source code aplikasi dapat diserahkan atau disimpan di repository yang disepakati.
3. Pihak Kedua tetap boleh menggunakan pengetahuan teknis umum, pola kode umum, dan pengalaman pengembangan untuk proyek lain.
4. Data penelitian, data pasien, dan materi khusus penelitian adalah milik Pihak Pertama.
5. Pihak Kedua tidak boleh mempublikasikan data pasien atau data rahasia tanpa izin Pihak Pertama.

## 14. Kerahasiaan Data

Pihak Kedua wajib menjaga kerahasiaan:

- Data pasien.
- Akun pengguna.
- Akses server.
- Akses database.
- Dokumen penelitian yang belum dipublikasikan.
- Informasi internal Pihak Pertama.

Pihak Pertama wajib memastikan penggunaan data pasien sudah sesuai izin etik, informed consent, dan kebijakan institusi.

## 15. Risiko dan Tanggung Jawab

Pihak Kedua bertanggung jawab pada pengembangan teknis aplikasi sesuai scope.

Pihak Kedua tidak bertanggung jawab atas:

- Kesalahan substansi instrumen yang telah disetujui Pihak Pertama.
- Penggunaan aplikasi tanpa izin etik.
- Kehilangan data akibat tidak adanya backup setelah akses diserahkan.
- Kerusakan server akibat perubahan pihak lain.
- Perubahan kebijakan hosting, domain, atau layanan pihak ketiga.

## 16. Penghentian Kerja Sama

Kerja sama dapat dihentikan apabila:

1. Salah satu pihak tidak memenuhi kewajiban utama.
2. Pembayaran tidak dilakukan sesuai termin.
3. Terjadi perubahan scope besar tanpa kesepakatan.
4. Para Pihak sepakat untuk mengakhiri kerja sama.

Pekerjaan yang sudah selesai tetap wajib dibayarkan sesuai progress.

## 17. Penyelesaian Perselisihan

Jika terjadi perselisihan, Para Pihak sepakat menyelesaikan terlebih dahulu melalui musyawarah. Jika tidak tercapai kesepakatan, penyelesaian dilakukan sesuai aturan hukum yang berlaku atau mekanisme yang disepakati Para Pihak.

## 18. Penutup

Dokumen ini dibuat sebagai dasar kerja sama pengembangan aplikasi. Hal-hal yang belum diatur dapat ditambahkan melalui addendum tertulis dan disetujui Para Pihak.
